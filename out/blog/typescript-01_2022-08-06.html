타입스크립트(TypeScript)란?

타입스크립트는 자바스크립트의 기본 문법에 타입을 부여하는 문법이 추가되어서 컴파일(Compille) 과정을 통해서 자바스크립트 코드로 변환되어 사용되는 정적 타입 언어(Static Type Language)입니다.


타입스크립트를 사용하는 이유

1. 에러의 사전 방지

타입스크립트의 타입선언을 통해서 에러를 사전에 예방할 수 있습니다.

function sum(a: number, b: number) {
  return a + b;
}

sum(2, 3); // 5
sum(2, "3"); // Error

자바스크립트에서는 2+”3” 이 암시적 형변환에 의해 문자열로 변환되어 “2”+”3” 가 되어 “23”이라는 결과가 나오게 됩니다. 개발자가 코드를 작성하며 실수를 하게 되면 의도하지 않은 결과가 일어날 수 있습니다.

타입스크립트에는 :을 이용해 정적으로 타입을 지정하는 타입 어노테이션(Type Annotation)이 있습니다. 코드 작성 시에 앞에서 선언했던 타입과 알맞지 않은 타입을 넣거나 타입을 넣지 않았을 경우 에러 메시지로 알려주기 때문에 컴파일 시점에서 에러를 잡아낼 수 있고 의도하지 않은 동작을 방지할 수 있습니다.


또한

2. 더 나은 개발자 경험

Visual Studio Code 같은 통합 개발 환경(IDE)에서 타입스크립트를 사용하면 변수에 마우스만 갖다 대도 데이터의 타입을 알 수 있고, 객체의 속성 값들을 자동으로 리스트해주기 때문에 코드 작성을 하는 것이 훨씬 편해집니다. 그리고 오류를 자동으로 수정해주는 '빠른 수정' 기능, 유용한 탐색 기능 등을 제공 해주기도 합니다.

또한 명시적으로 타입을 지정하기 때문에 코드를 작성할 때 개발자의 의도를 좀 더 직관적으로 표현할 수 있고, 마찬가지로 다른 사람의 코드를 읽을 때 더 명확하게 이해할 수 있습니다.
