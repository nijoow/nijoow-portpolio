스택(Stack)과 큐(Queue), 우선순위 큐(Priority Queue), 덱(Deque)

스택(Stack)

스택이란 후입선출(LIFO, Last In First Out)구조의 자료구조이다. 주로 스택의 시작을 밑(bottom), 끝을 위(top), 원소를 삽입하는 것을 푸시(push), 제거하는 것을 팝(pop) 이라고 부른다.

스택에서의 데이터처리

  • 데이터는 스택의 끝에만 삽입할 수 있음
  • 데이터는 스택의 끝에서만 삭제할 수 있음
  • 스택의 마지막 원소만 읽을 수 있음

사용되는 상황 : 마지막에 들어온 데이터부터 먼저 처리해야할 때

  • 브라우저의 방문기록(뒤로가기)
  • 실행 취소 (Ctrl + z)
  • 재귀 함수
  • 역순 문자열
  • 괄호 검사
  • 후위표기법 검사

큐(Queue)

큐는 선입선출(FIFO, First In First Out)구조의 자료구조이다. 주로 스택의 시작을 앞(front), 끝을 뒤(back/rear), 원소를 삽입하는 것을 인큐(enqueue), 제거하는 것을 디큐(dequeue) 라고 부른다.

큐에서의 데이터처리

  • 데이터는 큐의 끝에만 삽입할 수 있음
  • 데이터는 큐의 앞에서만 삭제할 수 있음
  • 큐의 앞에 있는 원소만 읽을 수 있음

사용되는 상황 : 데이터가 입력된 시간 순서대로 처리해야할 때

  • 너비 우선 탐색 (BFS) 알고리즘
  • 프로세스 관리
  • 우선순위가 같은 대기열

우선순위 큐(Priority Queue)

우선순위 큐는 각 요소가 값과 우선순위라는 2개의 데이터를 가지고 있는 자료구조이다. 요소를 제거할 때 우선순위가 높은 요소일수록 먼저 삭제되고, 우선순위가 같다면 먼저 들어온 순서대로 삭제된다.

사용되는 상황 :

  • 시뮬레이션
  • 네트워크 트래픽 제어
  • 운영 체제의 작업 스케줄링
  • 수치 해석적인 계산

일반적으로 힙(Heap)을 이용해 구현하고 항상 전체 트리가 최대 힙 구조를 유지하도록 해야한다.

힙(Heap)

힙은 여러 개의 값들 중에서 가장 큰 값이나 가장 작은 값을 빠르게 찾아내도록 만들어진 자료구조이다.

부모 노드의 키 값이 자식 노드보다 큰 경우 최대힙(max heap), 작은경우 최소힙(min heap)이라고 부른다.

힙의 요소들을 차례대로 번호를 붙이고 그 번호의 배열 인덱스에 힙의 노드들을 저장한다.

  • 왼쪽 자식의 인덱스 = (부모의 인덱스) * 2
  • 왼쪽 자식의 인덱스 = (부모의 인덱스) * 2 + 1
  • 부모의 인덱스 = (왼쪽 자식의 인덱스) / 2

우선순위 큐에서의 데이터처리

  • 삽입
  1. 완전 이진트리를 유지하는 형태로 순차적으로 삽입
  2. 루트 노드까지 올라가며 비교/교환하고 최대힙 구성
  • 삭제
  1. 루트 노드를 삭제하고 가장 마지막 원소를 루트 노드의 위치로 이동
  2. 리프 노드까지 내려가며 비교/교환하고 최대힙 구성

덱(Deque)

덱은 양쪽으로 데이터의 입출력이 가능한 자료구조로 Double ended Queue를 축약한 의미이다. 데이터의 삽입이나 삭제를 한쪽으로만 제한해둔 입력제한덱(Scroll), 출력제한덱(Shelp)도 존재한다.
