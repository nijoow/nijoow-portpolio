타입스크립트에서는 명명된 타입(named type)를 선언하는 방법은 타입 별칭(Type Aliases)과 인터페이스(interface)가 있습니다.

## 타입 별칭(Type Aliases)

객체 타입과 유니온 타입을 타입 어노테이션을 통해서 직접 작성할 수도 있지만 똑같은 유형의 타입을 여러번 작성하는 것은 불편할 것입니다. 같은 유형의 타입을 하나의 이름으로 참조할 수 있게하는 방법이 타입 별칭(Type Aliases)입니다.

```tsx
// 객체 타입
type NumObj = { x: number; y: number };

function add(numObj: NumObj) {
  return numObj.x + numObj.y;
}

function substract(xy: NumObj) {
  return numObj.x - numObj.y;
}

add({ x: 10, y: 5 });
substract({ x: 10, y: 5 });
```

```tsx
// 유니온 타입
type ID = number | string;
```

## 인터페이스(Interfaces)

인터페이스 선언은 객체 타입의 이름을 지정하는 또 다른 방법입니다. 기본적으로 타입 별칭과 인터페이스는 매우 유사하고 많은 경우에 자유롭게 선택해서 사용할 수 있습니다.

```tsx
interface NumObj {
  x: number;
  y: number;
}

function add(numObj: NumObj) {
  return numObj.x + numObj.y;
}

function substract(numObj: NumObj) {
  return numObj.x - numObj.y;
}

add({ x: 10, y: 5 });
substract({ x: 10, y: 5 });
```

## Type과 Interface의 차이점

가장 큰 차이점은 타입 별칭은 새로운 속성을 추가하기 위해 다시 선언할 수 없고, 인터페이스는 항상 확장 가능하다는 점입니다.

type은 `&`연산자, interface는 `extends`키워드를 통해 확장 가능합니다.

```tsx
// 타입 별칭
type Human = {
  name: string;
};

type Student = Human & {
  college: string;
};

// 인터페이스
interface Human {
  name: string;
}

interface Student extends Human {
  college: string;
}
```

interface는 동일한 이름으로 다시 선언해서 확장하는 것이 가능하지만 type은 불가능합니다.

```tsx
interface Human {
  name: string;
}

interface Human {
  age: number;
}
```
